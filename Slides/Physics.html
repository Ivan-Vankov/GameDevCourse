Triggers

Check for entering inside the space of a collider but don't apply any forces.

  • Doesn't behave like a solid object allows others to pass through it
  • Created with a Collider with isTrigger checkbox checked
  • OnTriggerEnter function is called inside scripts

Physics Layers

Allow better customization for collisions

  • Every GameObject belongs to a Physics Layer
  • We can customize which layer with which other layer will collider: Edit > Project Settings, then select the Physics

Colliders 3D

Component that is invisible shape of the GameObject used for collisions

  • Primitive colliders have better performance
  • GameObjects can have multiple collider components
  • Colliders can have Physics Material

Colliders 2D

  • Different Components from the 3D Colliders
  • Works in a separate 2D Physics Engine

RigidBody(3D)

Component that enables physical behaviour for a GameObject

  • It makes it move according to the collisions and its physical properties
  • Allows affection by gravity
  • Add properties like Mass and Drag

RigidBody2D

  • Different Component from RigidBody
  • Works in a separate 2D Physics Engine

Static Colliders

When the GameObject doesn't have a RigidBody

  • Static Colliders should be used for immovable objects. Don't change their transform position because that could cause errors in the physics engine!
  • Example usage: Ground Terrain, Buildings, immovable objects...

Kinematic Rigidbody Collider

When the GameObject has a RigidBody with "is Kinematic" checked

  • Use Colliders with Kinematic RigidBody to make objects which you don't want to react to physics but still move them through scripts or something.
  • Example usage: Sliding Doors, Traps...
